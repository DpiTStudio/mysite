# 🚀 DPIT-CMS: Полнофункциональная CMS на Django

Добро пожаловать в проект **DPIT-CMS**! Это мощная, современная и гибкая система управления контентом, специально разработанная для создания корпоративных сайтов, портфолио и систем управления услугами. Проект объединяет в себе лучшие практики разработки на Django с акцентом на автоматизацию процессов и удобство администрирования.

---

## 📂 Подробная структура проекта

Ниже представлен детальный обзор всех директорий и ключевых файлов проекта.

### 🏠 Корень проекта (`mysite/`)
- `README.md` — Этот файл, содержащий общий обзор системы.
- `manage.py` — Стандартный инструмент командной строки Django для управления проектом (миграции, запуск сервера).
- `requirements.txt` — Список всех зависимостей проекта (Django, Celery, Redis и др.).
- `.env` — Файл конфигурации переменных окружения (секреты, настройки БД, почты).
- `.gitignore` — Список файлов и папок, игнорируемых системой контроля версий Git.
- `db.sqlite3` — База данных SQLite (используется для разработки).

### ⚙️ Ядро системы (`mysite/mysite/`)
Центральная папка с настройками всего проекта:
- `settings.py` — Главный файл конфигурации (установленные приложения, настройки безопасности, БД, Celery).
- `urls.py` — Корневой файл маршрутизации, объединяющий пути всех приложений.
- `wsgi.py` / `asgi.py` — Интерфейсы для развертывания на боевых серверах (WSGI — стандартный, ASGI — асинхронный).
- `celery.py` — Конфигурация Celery для выполнения фоновых задач (например, бэкапов).

### 🛠 Приложения (Apps)
Каждое приложение отвечает за свой функционал:
- **[accounts/](./accounts/)** — Управление пользователями: расширенная модель, аутентификация, профили.
- **[main/](./main/)** — Ядро CMS: глобальные константы, утилиты переименования файлов, SEO-миксины.
- **[services/](./services/)** — Каталог услуг: управление ценами, категориями, тарифами и заявками.
- **[portfolio/](./portfolio/)** — Демонстрация работ: категории проектов, связь с услугами.
- **[news/](./news/)** — Модуль новостей: автоматическая генерация анонсов при добавлении контента.
- **[reviews/](./reviews/)** — Обратная связь: сбор и модерация отзывов клиентов.
- **[tickets/](./tickets/)** — Система поддержки: тикеты от пользователей и ответы администрации.
- **[cart/](./cart/)** — Корзина и заказы: оформление заявок на услуги с поддержкой промокодов.
- **[knowledge_base/](./knowledge_base/)** — База знаний: структурированные статьи и руководства (Вики).
- **[favorites/](./favorites/)** — Универсальная система "Избранного" на базе Generic FK.
- **[mail/](./mail/)** — Почтовый сервис: проверка IMAP, SMTP тесты, отправка уведомлений.
- **[logfiles/](./logfiles/)** — Управление логами: просмотр серверных логов прямо из админки.

### 📁 Ресурсные папки
- **[static/](./static/)** — Статические файлы: CSS-фреймворки, JS-библиотеки, глобальные стили.
- **[templates/](./templates/)** — Общие HTML-шаблоны: `base.html`, компоненты (`header`, `footer`, `breadcrumbs`).
- **[media/](./media/)** — Пользовательские загрузки: изображения портфолио, аватары, иконки услуг.
- **[backups/](./backups/)** — Автоматические резервные копии базы данных и медиа-файлов.
- **[logs/](./logs/)** — Журналы работы системы (ошибки, события Celery, запросы).

## 🛠 Технологический стек

- **Core**: Python 3.10+, Django 4.2+
- **Database**: PostgreSQL (Production) / SQLite (Dev)
- **Frontend**: Bootstrap 5, HTMX, Flatpickr, TinyMCE
- **Queue/Async**: Redis & Celery (для рассылок и бэкапов)
- **Security**: django-simple-captcha (защита форм), django-allauth
- **UI Admin**: [django-jazzmin](https://github.com/farridav/django-jazzmin) (премиальный вид админки)

## 🔧 Инструкции по развертыванию

1. **Подготовка окружения**:
   - Убедитесь, что установлен Python 3.10 и Redis.
   - Склонируйте репозиторий: `git clone <repo_url>`.

2. **Настройка переменных**:
   - Создайте файл `.env` в корне `mysite/`.
   - Заполните `SECRET_KEY`, `DEBUG`, настройки базы данных и почтовых серверов.

3. **Установка зависимостей**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Миграции и База Данных**:
   ```bash
   python manage.py migrate
   python manage.py createsuperuser
   ```

5. **Запуск проекта**:
   ```bash
   python manage.py runserver
   ```

6. **Фоновые задачи (Celery)**:
   ```bash
   celery -A mysite worker --loglevel=info
   celery -A mysite beat --loglevel=info
   ```

## ⚡ Особенности реализации
- **Единая SEO-система**: Каждая страница поддерживает управление Meta-тегами через миксины в `main`.
- **Авто-транслитерация**: Все загружаемые файлы автоматически переименовываются в латиницу с добавлением даты для избежания конфликтов.
- **Связанный контент**: Мощная система связей между услугами, портфолио и новостями.
