# Резервные копии (Backups)

Этот каталог предназначен для автоматического хранения архивных копий проекта: базы данных и медиа-файлов.

---

## 📂 Организация системы бэкапов

### 1. `db/` — Дампы базы данных
- Резервные копии SQLite или дампы PostgreSQL (в зависимости от настроек `.env`).
- Файлы именуются в формате: `db_backup_ГГГГ-ММ-ДД_ЧЧ-ММ.sql` или `.sqlite3.bak`.

### 2. `media/` — Архивы медиа-файлов
- Сжатые архивы (обычно `.tar.gz` или `.zip`) всей папки `media`.
- Имена: `media_backup_ГГГГ-ММ-ДД_ЧЧ-ММ.tar.gz`.

---

## ⚙️ Как работает автоматизация
Система бэкапов в **DPIT-CMS** работает на базе **Celery Beat**:
1. **Расписание**: Настраивается в `settings.py` (например, один раз в сутки).
2. **Задача**: Celery worker выполняет задачу `create_backup` из файла `mysite.backup_tasks`.
3. **Хранение**: В этой папке хранятся последние несколько копий (настраиваемый параметр ротации).
4. **Очистка**: Старые архивы старше определенного срока (например, более 30 дней) автоматически удаляются для экономии места на диске.

---

## 🛠 Ручное создание и восстановление
- **Создать бэкап сейчас**:
  ```bash
  python manage.py shell
  >>> from mysite.backup_tasks import perform_full_backup
  >>> perform_full_backup.delay()
  ```
- **Восстановление**:
    - Для восстановления базы данных используйте системные команды:
        - SQLite: Просто скопируйте файл поверх текущего `db.sqlite3`.
        - PostgreSQL: `psql -U username -d db_name < backup_file.sql`.
    - Для восстановления медиа: Разархивируйте содержимое архива в папку `media`.

---

## ⚠️ Важные рекомендации
- **Внесерверное хранение**: Никогда не храните бэкапы *только* на том же сервере, где работает сайт. Настройте автоматическую выгрузку файлов из этой папки в облачное хранилище (S3, Dropbox, Yandex.Disk).
- **Проверка**: Периодически скачивайте бэкапы и проверяйте их на работоспособность, чтобы убедиться в отсутствии проблем при создании копий.
