# Документация приложения Cart (Корзина и Заказы)

Приложение `cart` реализует функционал корзины покупок, оформление заказов на услуги и работы, систему промокодов и автоматическую регистрацию клиентов.

## Обзор
Это приложение позволяет пользователям собирать интересующие их услуги и проекты в единый список (корзину), применять скидки по промокодам и оформлять полноценные заказы. Особенностью является механизм "бесшовной" покупки: если гость оформляет заказ, система автоматически создает ему учетную запись и отправляет данные для входа на email.

## Описание файлов

### 1. `cart.py`
**Логика корзины (не модель БД).**
- Хранит данные корзины в текущей сессии пользователя (`request.session`).
- Методы: `add` (добавление), `remove` (удаление), `save` (сохранение в сессию), `clear` (очистка), а также расчет итоговой стоимости.

### 2. `admin.py`
Настройка административной панели.
- `OrderAdmin`: Управление заказами. Позволяет видеть состав заказа (OrderItem) через инлайны, менять статусы и просматривать данные о применении промокодов.
- `PromoCodeAdmin`: Создание и настройка промокодов (тип скидки, сроки действия, лимиты использований).
- `PromoCodeUsageAdmin`: История активаций промокодов пользователями.

### 3. `apps.py`
Конфигурация приложения.
- `verbose_name = "Корзина и Заказы"`: Имя приложения в админке.

### 4. `forms.py`
Формы приложения.
- `OrderCreateForm`: Форма оформления заказа (Имя, Фамилия, Email, Телефон, Комментарий). Включает чекбокс "Создать личный кабинет" для гостей.

### 5. `models.py`
Структура данных.

**Классы:**

#### `Order`
Основная информация о заказе.
- **Поля:** Клиентские данные, привязка к `User` (если авторизован), статус (Новый, В работе, Завершен и др.), флаг авторегистрации, примененный промокод и сумма скидки.

#### `OrderItem`
Позиция внутри заказа.
- **Поля:** Ссылка на `Order`, ссылка на `Service` или `Portfolio`, цена на момент заказа (фиксация стоимости), количество.

#### `PromoCode`
Модель скидочного купона.
- **Поля:** Код (строка), тип (Процент или Фикс. сумма), значение, сроки действия, лимиты активаций.

#### `PromoCodeUsage`
Запись о факте использования конкретного промокода в заказе конкретным пользователем.

### 6. `views.py`
Логика преставлений.
- `cart_add` / `cart_remove`: Управление содержимым сессионной корзины (HTMX-поддержка).
- `cart_detail`: Страница просмотра корзины.
- `order_create`: Главный процесс оформления. Реализует:
    1. Валидацию данных клиента.
    2. Применение промокода из сессии.
    3. Автоматическую регистрацию клиента (генерация пароля, создание `User`).
    4. Сохранение заказа и очистку корзины.
    5. Отправку Email-подтверждения с данными заказа (и логином/паролем для новых пользователей).
- `order_success`: Страница подтверждения успешного заказа.
- `apply_promo` / `remove_promo`: AJAX-интерфейсы для применения купонов на лету (без перезагрузки страницы).

### 7. `urls.py`
Маршруты URL.
- `detail/` -> Просмотр корзины.
- `add/<str:item_type>/<int:item_id>/` -> Добавление в корзину.
- `remove/...` -> Удаление из корзины.
- `checkout/` -> Оформление заказа.
- `success/<int:order_id>/` -> Успех.
- `apply-promo/` -> API промокода.
