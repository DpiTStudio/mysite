# Документация приложения Knowledge Base (База знаний)

Приложение `knowledge_base` реализует функционал Вики-системы или справочника для хранения обучающих статей, документации и ответов на частые вопросы.

## Обзор
Приложение позволяет структурировать контент по категориям, отслеживать популярность статей через счетчик просмотров и оформлять тексты с использованием полноценного HTML-редактора. Поддерживает SEO-параметры и динамическую генерацию "хлебных крошек" для удобной навигации.

---

## Описание файлов

### 1. `admin.py`
Настройка административной панели.
- `CategoryAdmin`: Управление рубриками базы знаний. Позволяет задавать иконки и настраивать порядок отображения.
- `ArticleAdmin`: Основное управление контентом статей. Обеспечивает фильтрацию по категориям и статусу публикации.

### 2. `apps.py`
Конфигурация приложения.
- `verbose_name = "База знаний"`: Человекочитаемое имя приложения.

### 3. `models.py`
Структура данных.

**Классы:**

#### `Category`
Раздел базы знаний (например: "Инструкции", "FAQ", "Полезные статьи").
- **Поля:** Иконка, описание (HTML), порядок сортировки, статистика просмотров категории.
- **Методы:** `get_icon_admin` (превью иконки в админке), `get_absolute_url`.

#### `Article`
Информационная статья.
- **Поля:** Привязка к `Category`, заголовок, контент (TinyMCE), слаг для URL, счетчик просмотров, флаг публикации.
- **Методы:** Автоматическое отслеживание даты создания и обновления.

### 4. `views.py`
Логика преставлений.
- `index`: Главная страница базы знаний. Отображает список всех категорий и 5 наиболее свежих статей.
- `category_detail`: Список статей внутри выбранной категории.
- `article_detail`: Полнотекстовое отображение статьи. При каждом просмотре инкрементирует счетчик `views`. Формирует цепочку хлебных крошек.

### 5. `urls.py`
Маршруты URL.
- `/` -> Главная страница базы знаний.
- `category/<slug>/` -> Страница категории.
- `article/<slug>/` -> Страница статьи.

---

## Особенности реализации
1. **TinyMCE**: Использование визуального редактора позволяет вставлять в статьи видео, изображения, таблицы и форматированный код.
2. **SEO Friendly**: Каждый раздел и статья имеют уникальные URL-адреса (слаги) и автоматическое управление заголовками страниц.
3. **Навигация**: Система хлебных крошек облегчает пользователю возврат в корень базы знаний из глубоко вложенных статей.
