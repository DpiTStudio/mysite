# Приложение Logfiles

Модуль для мониторинга, управления и резервного копирования системных логов Django прямо через административную панель.

## Файлы приложения

### `models.py`
- `LogFile` — представляет файл лога в системе. Хранит путь к файлу, размер и дату последнего изменения. Имеет метод `update_file_info()` для синхронизации данных с реальным файлом на диске.
- `LogBackup` — хранит информацию о созданных резервных копиях логов. Сами копии загружаются в `logs/backups/`.

### `admin.py`
Содержит основную логику управления логами через Django Admin:
- Просмотр содержимого лог-файлов.
- Создание резервных копий (backup).
- Очистка файлов логов.
- Специальные представления (views) для работы с файлами.

### `management/commands/`
- `scan_logs.py` — команда для автоматического поиска и регистрации лог-файлов из директории логов в базу данных.

### `views.py`
В данной версии приложения основная логика вынесена в `admin.py`, но файл может содержать заглушки или специфические представления.

## Основные возможности
1. **Автоматическое сканирование**: Поиск всех `.log` файлов в проекте.
2. **Чтение в реальном времени**: Просмотр последних строк лога через админку.
3. **Безопасность**: Возможность сделать "бэкап" перед очисткой текущего лога.
4. **Удобство**: Полный контроль над журналом событий без доступа по SSH.
