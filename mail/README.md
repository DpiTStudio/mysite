# Документация приложения Mail

Приложение `mail` предназначено для проверки и использования почтового функционала сайта. Оно позволяет просматривать последние входящие письма (через IMAP) и отправлять тестовые письма (через SMTP) администраторам сайта.

## Обзор
Приложение не имеет собственных моделей базы данных. Оно работает напрямую с почтовыми протоколами, используя настройки из `settings.py`. Доступ к функционалу ограничен правами персонала (`staff`).

---

## Описание файлов

### 1. `apps.py`
Файл конфигурации приложения.
- `verbose_name = 'Управление почтой'`: Имя в админке.

### 2. `urls.py`
Файл маршрутизации.
- `app_name = "mail"`: Пространство имен URLs.
- **Маршруты:**
    - `""` -> `views.mail_index`: Главная страница почтового интерфейса.
    - `send-test/` -> `views.send_test_email`: Обработчик отправки тестового письма.

### 3. `views.py`
Основная логика работы с почтой.

**Функции представлений:**

#### `mail_index(request)`
Отображает последние входящие письма.
1. **Подключение к IMAP:** Использует настройки из `settings.py`.
2. **Получение писем:** Загружает заголовки последних 10 писем.
3. **Обработка данных:** Декодирует тему и отправителя.
4. **Рендер:** Шаблон `mail/index.html`.

#### `send_test_email(request)`
Обрабатывает отправку тестового письма.
- Использует функцию Django `send_mail`.
- Отправляет статический текст на указанный адрес.

### 4. `models.py` и `admin.py`
Файлы пусты, так как приложение не хранит данные в базе данных.
