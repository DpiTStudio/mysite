# Пользовательские файлы (Media)

Этот каталог содержит все загруженные пользователями и администраторами медиа-файлы: изображения для новостей, портфолио, аватары и иконки.

---

## 📂 Организация по приложениям и типам данных

### 1. `portfolio/` — Изображения для проектов
- `images/`: Главные фото выполненных работ.
- `categories/`: Логотипы и обложки категорий портфолио.
- `thumbnails/`: Автоматически сгенерированные превью (если используется миниатюризация).

### 2. `news/` — Контент новостей
- Изображения для статей, анонсов и фотогалерей к новостям.
- Файлы организованы по годам/месяцам загрузки (рекомендуемая практика).

### 3. `accounts/` — Профили пользователей
- `avatars/`: Фотографии профилей пользователей сайта.
- Все изображения автоматически переименовываются по шаблону `username_date`.

### 4. `services/` — Услуги
- `icons/`: Иконки для каждой услуги (SVG/PNG).
- `categories/`: Обложки и иконки групп услуг.

### 5. `logos/` и `favicons/` — Брендирование
- `logos/`: Варианты логотипа для темной и светлой шапки.
- `favicons/`: Набор значков сайта для различных устройств и браузеров.

### 6. `fon_headers/` — Оформление страниц
- Здесь хранятся широкоформатные фоновые изображения для верхнего блока (Header) разделов сайта.

---

## ⚙️ Автоматизация и управление файлами
- **RenameUploadTo**: В проекте используется специальная утилита во всех моделях, которая автоматически переименовывает загруженные файлы. Это предотвращает:
    1. Конфликты одинаковых имен файлов.
    2. Проблемы с кириллицей в путях URL.
    3. Разрастание папки "свалки" (все файлы попадают в соответствующие подпапки).
- **SEO-оптимизация**: Имена файлов формируются на основе заголовков (Title) объектов для лучшего ранжирования в поиске картинок.

---

## 🛠 Обслуживание и безопасность
- В режиме разработки (`DEBUG=True`) файлы отдаются самим Django.
- В режиме продакшена раздача медиа-файлов должна быть настроена через веб-сервер (Nginx или Apache).
- Для защиты рекомендуется использовать миграции и регулярные бэкапы папки `media`.
- **Никогда** не храните критически важные секреты в этой папке, так как она доступна публично.
