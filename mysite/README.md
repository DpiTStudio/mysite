# Конфигурация проекта (mysite)

Этот каталог содержит основные системные настройки Django-проекта `dpit-cms`.

## Состав каталога

### 1. `settings.py`
Центральный файл настроек проекта.
- **Безопасность:** Интеграция с `.env` (через `django-environ`).
- **Приложения:** Подключение встроенных (Admin, Auth) и кастомных приложений проекта.
- **Интерфейс:** Тонкая настройка `Jazzmin` (темы, иконки, меню админки).
- **Файлы:** Настройки статики (`STATIC_ROOT`) и медиа (`MEDIA_ROOT`).
- **Почта:** Конфигурация SMTP и IMAP.
- **Очереди:** Настройки Celery и Redis.

### 2. `urls.py`
Главный файл маршрутизации.
- Подключает URL всех приложений проекта (`news`, `portfolio`, `accounts` и др.).
- Обрабатывает системные пути: `admin/`, `captcha/`, `sitemap.xml`, `robots.txt`.
- Содержит настройки для обслуживания медиа-файлов.

### 3. `celery.py`
Файл инициализации Celery. Настраивает брокер сообщений для выполнения фоновых задач.

### 4. `backup_tasks.py`
Автоматизация резервного копирования.
- Содержит Celery-задачи для создания бэкапов базы данных и архивации медиа-файлов.
- Логика очистки старых архивов.

### 5. `context_processors.py`
Системные процессоры контекста.
- Например, `season_theme` для динамической смены оформления сайта в зависимости от времени года.

### 6. `wsgi.py` / `asgi.py`
Точки входа для WSGI (традиционные сервера) и ASGI (асинхронные сервера) интерфейсов.

---

## Важные настройки в `settings.py`:
- `DEBUG`: Режим отладки (должен быть `False` в продакшене).
- `ALLOWED_HOSTS`: Список разрешенных доменов.
- `AUTH_USER_MODEL`: Использование кастомной модели пользователя `accounts.User`.
- `LOGGING`: Конфигурация записи логов в файлы и консоль.
