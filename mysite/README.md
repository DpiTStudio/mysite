# ⚙️ Системная конфигурация проекта (mysite/mysite)

Этот каталог является "мозговым центром" всего Django-проекта **DPIT-CMS**. Здесь сосредоточены все глобальные настройки, маршрутизация и системные скрипты инициализации.

---

## 📂 Описание файлов и их назначения

### 1. `settings.py` — Главный файл настроек
Это сердце проекта, где определены все параметры работы:
- **Безопасность**: Загрузка секретных ключей, настроек БД и почты из файла `.env` (через `django-environ`).
- **Middleware**: Настройка промежуточных слоев (сессии, CSRF, Clickjacking protection).
- **Приложения (INSTALLED_APPS)**: Подключение системных и кастомных приложений проекта.
- **Интерфейс админки (Jazzmin)**: Конфигурация темы, иконок, меню и брендирования панели управления.
- **Хранилище (Static & Media)**: Пути к статическим файлам и пользовательскому контенту.
- **Интеграция очередей**: Настройки брокера сообщений Redis для работы с Celery.

### 2. `urls.py` — Главный диспетчер URL
Файл, определяющий структуру адресов всего сайта:
- **Маршрутизация приложений**: Подключает `urls.py` каждого модуля (`portfolio`, `news`, `accounts` и др.).
- **Системные пути**: Админ-панель (`admin/`), капча (`captcha/`), TinyMCE.
- **SEO-ресурсы**: Генерация `sitemap.xml` и обслуживание `robots.txt`.
- **Медиа-файлы**: Настройка раздачи изображений в режиме отладки (`DEBUG=True`).

### 3. `celery.py` — Настройка фоновых задач
Инициализация Celery:
- Связывает Django с Redis.
- Автоматически находит задачи (`tasks.py`) во всех приложениях проекта.

### 4. `backup_tasks.py` — Система резервного копирования
Скрипты для автоматизации сохранности данных:
- **Бэкап БД**: Создание дампов базы данных SQLite/PostgreSQL.
- **Бэкап медиа**: Архивация папки со всеми загруженными изображениями.
- **Celery Beat**: Интеграция с планировщиком задач для запуска копирования по расписанию.

### 5. `context_processors.py` — Глобальные данные в шаблонах
Функции, передающие данные во все HTML-шаблоны без необходимости прописывать их в каждом `view`:
- Например, передача настроек сайта или системных флагов.

### 6. `wsgi.py` и `asgi.py` — Интерфейсы развертывания
- `wsgi.py`: Для классических серверов (Apache, Gunicorn).
- `asgi.py`: Для асинхронных серверов (Daphne, Uvicorn) и поддержки WebSockets.

---

## 🔑 Ключевые параметры в `settings.py`

- `AUTH_USER_MODEL`: Указывает на `accounts.User`, заменяя стандартную модель пользователя Django.
- `CACHES`: Настройка кэширования (обычно через Redis).
- `LOGGING`: Описывает уровни логирования (INFO, ERROR) и пути сохранения файлов журналов.
- `CSRF_TRUSTED_ORIGINS`: Список доверенных доменов для защиты от межсайтовой подделки запросов.
- `CAPTCHA_xxx`: Настройки внешнего вида и сложности капчи на формах.
