# Документация приложения Portfolio

Приложение `portfolio` отвечает за управление портфолио (выполненными работами) и кейсами компании.

## Обзор
Это приложение позволяет демонстрировать выполненные проекты, группировать их по категориям и отслеживать популярность через счетчик просмотров. Каждая работа может быть настроена как "доступная для заказа", что интегрирует её с системой корзины.

---

## Описание файлов

### 1. `admin.py`
Настройка административной панели.

**Классы:**
- `PortfolioCategoryAdmin`: Управление категориями портфолио. Поддерживает кастомизацию SEO-параметров и уникальные шапки (Header) для каждой категории.
- `PortfolioAdmin`: Управление работами.
    - **Особенности:**
        - Отображение миниатюр изображений.
        - Быстрое управление флагом "Доступно для заказа".
        - Группировка полей: Основное, Ценообразование (для заказов), SEO.

### 2. `apps.py`
Конфигурация приложения.
- `verbose_name = "Портфолио"`: Человекочитаемое имя приложения в панели управления.

### 3. `context_processors.py`
Глобальные данные для шаблонов.
- `latest_portfolio(request)`: Возвращает список последних выполненных работ для отображения в футере или специальных блоках на любой странице сайта.

### 4. `forms.py`
Формы приложения.
- `PortfolioForm` / `PortfolioCategoryForm`: Интегрируют визуальный редактор **TinyMCE** для удобного форматирования описаний кейсов.

### 5. `models.py`
Структура данных.

**Классы:**

#### `PortfolioCategory(ActiveModel, SEOModel, HeaderModel)`
Категория работ (например, "Веб-дизайн", "Мобильные приложения").
- **Поля:** Логотип, описание, порядок сортировки.

#### `Portfolio(ActiveModel, SEOModel, TimestampModel)`
Конкретный проект в портфолио.
- **Ключевые поля:**
    - `category`: Привязка к категории.
    - `image`: Главное изображение проекта.
    - `content`: Подробное описание (HTML).
    - `views`: Счетчик просмотров.
    - **Параметры заказа:**
        - `is_available_for_order`: Флаг возможности добавить работу в корзину (как готовое решение).
        - `price_type`: Тип цены (Фикс, Диапазон, По договоренности).
        - `price`, `price_min`, `price_max`: Численные значения стоимости.
- **Методы:**
    - `increment_views()`: Безопасное увеличение счетчика просмотров.
    - `get_price_display()`: Форматирование стоимости для вывода пользователю.

### 6. `views.py`
Логика отображения.
- `portfolio_list(request)`: Список всех работ с пагинацией и фильтрацией по активности.
- `portfolio_by_category(request, category_slug)`: Список работ в конкретной категории.
- `portfolio_detail(request, slug)`: Страница конкретного кейса. Увеличивает счетчик просмотров при посещении и подбирает "Похожие работы".

### 7. `urls.py`
Маршруты URL.
- `/` -> Список всех работ.
- `/category/<slug>/` -> Фильтр по категории.
- `/<slug>/` -> Детальная страница работы.
