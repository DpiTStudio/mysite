# Документация приложения Services

Приложение `services` управляет каталогом услуг компании, их ценами, тарифами и формами заказа.

## Обзор
Это приложение является ключевым для бизнес-логики сайта. Оно позволяет структурировать предложения компании, задавать гибкие условия ценообразования (фиксы, диапазоны, договоренности) и предлагать клиентам выбор тарифных планов. Каждая услуга может быть привязана к примерам работ из приложения `portfolio`.

---

## Описание файлов

### 1. `admin.py`
Настройка административной панели.

**Классы:**
- `TechnologyAdmin`: Управление списком технологий (стеком).
- `ServiceCategoryAdmin`: Управление категориями услуг (Иконки, SEO, Сортировка).
- `ServiceAdmin`: Основное управление услугами.
    - **Инлайны (Inline Admin):**
        - `ServiceBenefitInline`: Преимущества услуги.
        - `ServiceStepInline`: Этапы работы.
        - `ServiceFAQInline`: Часто задаваемые вопросы.
        - `ServicePricePlanInline`: Тарифные планы.
- `ServiceOrderAdmin`: Просмотр и управление заявками от клиентов. Поддерживает изменение цветовых статусов.

### 2. `apps.py`
Конфигурация приложения.
- `verbose_name = "Услуги"`: Человекочитаемое имя приложения.

### 3. `forms.py`
Формы приложения.
- `ServiceOrderForm`: Форма обратной связи для заказа конкретной услуги (Имя, Телефон, Email, Сообщение).
- `ServiceAdminForm`: Подключает TinyMCE для админки.

### 4. `models.py`
Структура данных.

**Классы:**

#### `Technology`
Справочник технологий (например: Python, Django, React, FIGMA).

#### `ServiceCategory(ActiveModel, SEOModel, TimestampModel)`
Категория услуг (например: "Разработка сайтов", "Маркетинг").

#### `Service(ActiveModel, SEOModel, TimestampModel)`
Сама услуга.
- **Поля:** Иконка, Краткое и полное описание.
- **Ценообразование:**
    - `price_type`: Фиксированная, От и До, По договоренности.
    - `price_fixed`, `price_min`, `price_max`: Численные значения.
    - `currency`: Валюта (RUB, USD, EUR, KZT).
- **Связи:** ManyToMany связь с `Technology` и `Portfolio`.
- **Сложность:** Уровни (Простой, Средний, Сложный, Ультра-кодинг).

#### `ServiceOrder(TimestampModel)`
Заявка на услугу (лид). Собирает данные клиента и выбранную услугу/тариф.

#### `ServiceBenefit`, `ServiceStep`, `ServiceFAQ`
Дополнительные блоки контента для страницы услуги: преимущества, шаги реализации и ответы на вопросы.

#### `ServicePricePlan`
Тарифные планы для услуги (например: "Старт", "Бинес", "VIP"). Позволяет задать список включенных возможностей.

### 5. `signals.py`
Содержит сигналы, обрабатывающие события сохранения услуг или заказов (например, уведомления администратору).

### 6. `views.py`
Логика преставлений.
- `service_list(request)`: Каталог услуг с фильтрацией по категориям и сортировкой по цене/популярности.
- `service_detail(request, slug)`: Полноценный лендинг услуги со всеми блоками (преимущества, этапы, отзывы, похожие работы) и формой заказа.

### 7. `urls.py`
Маршруты URL.
- `/` -> Каталог всех услуг.
- `/<slug>/` -> Детальная страница услуги.
- `/order/<int:pk>/` -> Прямой заказ услуги.
