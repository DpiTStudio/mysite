# Документация приложения Services

Приложение `services` отвечает за управление каталогом услуг, тарификацию и обработку заказов на услуги.

## Обзор
Приложение предоставляет расширенный функционал для представления услуг компании. Основные возможности:
- Гибкое управление ценами (фиксированные, диапазоны "от и до", договорные).
- Технические требования (стек технологий) для каждой услуги.
- Система оформления заказов с уведомлениями.
- Поиск по услугам и фильтрация по категориям и сложности.
- Учет просмотров каждой услуги.

---

## Описание файлов

### 1. `admin.py`
Настройка административной панели.
- `ServiceAdmin`: Управление услугами. Поддерживает визуальное редактирование через TinyMCE, выбор технологий чекбоксами, управление SEO-тегами.
- `ServiceOrderAdmin`: Обработка заявок. Включает экспорт в CSV, цветное отображение статусов и компактный вывод контактов.

### 2. `apps.py`
Конфигурация приложения.
- `verbose_name = "Услуги"`: Имя в админке.

### 3. `forms.py`
Формы приложения.
- `ServiceAdminForm`: Форма для админки с интеграцией TinyMCE и удобным выбором технологий.
- `ServiceOrderForm`: Публичная форма заказа услуги (ФИО, Телефон, Email, Сообщение).

### 4. `models.py`
Структура данных.

**Классы:**
- `Service`: Основная модель услуги. Содержит поля для описания, иконок, цен, категорий и технических требований. Включает логику валидации цен и форматирования их отображения.
- `ServiceOrder`: Модель заказа. Хранит информацию о клиенте, выбранной услуге, статусе заказа и заметках администратора.

### 5. `views.py`
Логика представлений.
- `ServiceListView`: Список всех активных услуг с пагинацией и фильтрами.
- `ServiceDetailView`: Детальная страница услуги с формой заказа и блоком похожих услуг.
- `ServiceOrderView`: Обработчик отправки формы заказа.
- `ServiceSearchView`: Поиск по заголовкам, описаниям и технологиям.

### 6. `urls.py`
Маршруты URL.
- `/` -> Список услуг.
- `/search/` -> Поиск.
- `/<slug>/` -> Детальная страница.
- `/<slug>/order/` -> Обработка заказа.
