# Выходная папка статики (StaticFiles)

Этот каталог (`staticfiles`) создается автоматически командой `collectstatic` и содержит все собранные воедино статические файлы из всех приложений проекта.

---

## ⚙️ Назначение каталога
В Django-проектах статические ресурсы (CSS, JS, изображения) могут быть разбросаны по папкам приложений. Для эффективной отдачи файлов в реальном окружении (продакшн) веб-сервер (например, Nginx) ожидает увидеть их в одном месте. 

**Этот каталог не предназначен для ручного редактирования!** Все изменения нужно вносить в исходные папки `static` в корне или внутри приложений.

---

## 🛠 Процесс сборки
Для обновления содержимого этой папки выполните команду:
```bash
python manage.py collectstatic
```

### Что происходит при выполнении:
1. Django обходит `INSTALLED_APPS` и находит все папки `static`.
2. Копирует их содержимое в этот каталог.
3. Собирает файлы из системных каталогов (например, стили админки, TinyMCE, Jazzmin).
4. Создает (при настройке) манифест хэшей файлов для управления кэшированием.

---

## 📂 Приблизительный состав (после сборки)
- **`admin/`**: Дизайн стандартной панели управления Django.
- **`jazzmin/`**: Ресурсы темы оформления административной панели.
- **`tinymce/`**: Скрипты и стили визуального редактора контента.
- **`rest_framework/`**: (Если используется API) Ресурсы интерактивного интерфейса API.
- **`css/`, `js/`, `images/`**: Ваши кастомные файлы из основной папки `static`.

---

## ⚠️ Важно для продакшена
- При использовании библиотеки `WhiteNoise` или раздаче через Nginx, этот каталог является источником данных.
- Не включайте этот каталог в систему контроля версий (Git), исключите его через `.gitignore`, так как он является артефактом сборки.
