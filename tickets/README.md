# Приложение Tickets

Модуль технической поддержки для авторизованных пользователей. Позволяет создавать обращения (тикеты) и вести переписку с администрацией.

## Файлы приложения

### `models.py`
- `Ticket` — основная модель обращения. Содержит:
    - `user` (автор тикета)
    - `subject` (тема)
    - `description` (описание проблемы)
    - `status` (Открыт, В работе, Решен, Закрыт)
    - `priority` (Низкий, Средний, Высокий, Срочный)
- `TicketMessage` — сообщения внутри тикета. Каждое сообщение привязано к тикету и пользователю (автору).

### `views.py`
- `ticket_list(request)` — список тикетов. Обычные пользователи видят только свои тикеты, персонал — все тикеты системы. Поддерживается фильтрация по статусу.
- `ticket_create(request)` — форма создания нового обращения.
- `ticket_detail(request, ticket_id)` — детальная страница тикета со всей историей переписки и формой ответа. Если пользователь пишет в закрытый тикет, его статус автоматически меняется на "Открыт".
- `ticket_close(request, ticket_id)` — функционал закрытия тикета.

### `forms.py`
- `TicketForm` — форма создания тикета.
- `TicketMessageForm` — форма для отправки сообщения в существующий тикет.

### `urls.py`
Маршруты для управления тикетами (список, создание, просмотр, закрытие).

### `templates/tickets/`
- `list.html` — список обращений пользователя.
- `create.html` — форма нового тикета.
- `detail.html` — чат поддержки и информация о статусе.
