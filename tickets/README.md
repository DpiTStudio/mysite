# Документация приложения Tickets

Приложение `tickets` предназначено для системы технической поддержки, позволяя пользователям создавать обращения и общаться с администраторами.

## Обзор
Приложение реализует полноценную систему тикетов с возможностью создания обращений, добавления сообщений, отслеживания статусов и приоритетов. Доступ к тикетам разграничен: обычные пользователи видят только свои тикеты, персонал видит все.

---

## Описание файлов

### 1. `admin.py`
Настройка административной панели.
- `TicketAdmin`: Управление тикетами (статус, приоритет, пользователь).
- `TicketMessageAdmin`: Управление сообщениями в тикетах.

### 2. `models.py`
Структура данных.

**Классы:**
- `TicketStatus`: Перечисление статусов (Открыт, В работе, Решен, Закрыт).
- `TicketPriority`: Перечисление приоритетов (Низкий, Средний, Высокий, Срочный).
- `Ticket`: Модель тикета.
- `TicketMessage`: Модель сообщения в тикете.

### 3. `forms.py`
Формы приложения.
- `TicketForm`: Форма создания тикета с капчей.
- `TicketMessageForm`: Форма добавления сообщения.

### 4. `views.py`
Логика представлений.

**Функции:**
- `ticket_list`: Список тикетов.
- `ticket_create`: Создание нового тикета.
- `ticket_detail`: Детальная страница тикета и добавление сообщений.
- `ticket_close`: Закрытие тикета.

### 5. `urls.py`
Маршруты URL.
- `/` -> Список тикетов.
- `/create/` -> Создание тикета.
- `/<int:ticket_id>/` -> Детальная страница.
- `/<int:ticket_id>/close/` -> Закрытие тикета.
