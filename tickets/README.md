# Документация приложения Tickets

Приложение `tickets` предназначено для системы технической поддержки, позволяя пользователям создавать обращения и общаться с администраторами.

## Обзор
Приложение реализует полноценную систему тикетов с возможностью создания обращений, добавления сообщений, отслеживания статусов и приоритетов. Доступ к тикетам разграничен: обычные пользователи видят только свои тикеты, персонал видит все.

---

## Описание файлов

### 1. `admin.py`
Настройка административной панели.

**Классы:**
- `TicketAdmin`: Управление тикетами.
    - Отображение: ID, тема, пользователь, статус, приоритет, даты.
    - Фильтры по статусу, приоритету и дате создания.
    - Группировка полей (Основная информация, Статус и приоритет, Даты).
- `TicketMessageAdmin`: Управление сообщениями в тикетах.
    - Отображение: ID, тикет, автор, дата, флаг внутреннего сообщения.

### 2. `models.py`
Структура данных.

**Классы:**

#### `TicketStatus(models.TextChoices)`
Перечисление статусов тикета:
- `OPEN`: Открыт
- `IN_PROGRESS`: В работе
- `RESOLVED`: Решен
- `CLOSED`: Закрыт

#### `TicketPriority(models.TextChoices)`
Перечисление приоритетов:
- `LOW`: Низкий
- `MEDIUM`: Средний
- `HIGH`: Высокий
- `URGENT`: Срочный

#### `Ticket(TimestampModel)`
Модель тикета.
- **Поля:** Пользователь, тема, описание, статус, приоритет, дата закрытия.

#### `TicketMessage(TimestampModel)`
Модель сообщения в тикете.
- **Поля:** Тикет, автор, текст сообщения, флаг внутреннего сообщения.

### 3. `forms.py`
Формы приложения.

- `TicketForm`: Форма создания тикета с капчей. Поля: тема, описание, приоритет.
- `TicketMessageForm`: Форма добавления сообщения в тикет.

### 4. `views.py`
Логика представлений.

**Функции:**

#### `ticket_list(request)`
Список тикетов пользователя (для персонала — все тикеты).
- Фильтрация по статусу через GET-параметр.
- Пагинация по 10 тикетов на страницу.

#### `ticket_create(request)`
Создание нового тикета.
- Форма с капчей.
- Автоматически привязывает тикет к текущему пользователю.

#### `ticket_detail(request, ticket_id)`
Детальная страница тикета.
- Отображает все сообщения в тикете.
- Позволяет добавлять новые сообщения.
- Автоматически переоткрывает закрытый тикет при добавлении нового сообщения.

#### `ticket_close(request, ticket_id)`
Закрытие тикета.
- Устанавливает статус `CLOSED` и сохраняет дату закрытия.

### 5. `urls.py`
Маршруты URL.
- `/` -> Список тикетов.
- `/create/` -> Создание тикета.
- `/<int:ticket_id>/` -> Детальная страница тикета.
- `/<int:ticket_id>/close/` -> Закрытие тикета.
